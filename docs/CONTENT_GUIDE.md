# 内容维护指南

本文档用于说明如何更新榄雕云艺项目中的文字内容、图片资源和展示数据，适合后续维护仓库时直接参考。

## 内容来源

项目当前主要内容来源于两个位置：

1. `assets/data/content.json`
2. `assets/images/`

其中：

- `content.json` 负责存放文字结构数据
- `assets/images/` 负责存放展品图片

## content.json 结构说明

`assets/data/content.json` 当前包含 4 个主要字段：

### 1. `introPages`

用于启动页后的引导页内容。

每一项包含：

- `title`：引导页标题
- `subtitle`：引导页说明
- `note`：强调信息

示例：

```json
{
  "title": "数字展馆",
  "subtitle": "通过高清图像与细节放大，靠近每一处窗格、人物与刀法纹理。",
  "note": "Hero 过渡 · 缩放浏览 · 新中式留白"
}
```

### 2. `featuredQuote`

用于引导页底部展示的一段强调语句。

### 3. `exhibits`

用于数字展馆中的作品列表。

每件展品包含：

- `id`：唯一标识，不能重复
- `title`：作品名称
- `author`：作者
- `era`：年代
- `category`：分类
- `technique`：工艺技法
- `description`：作品简介
- `story`：延伸说明
- `image`：图片资源路径

示例：

```json
{
  "id": "exhibit_1",
  "title": "东坡夜游",
  "author": "陈祖章",
  "era": "清代",
  "category": "人物",
  "technique": "圆雕、镂空雕",
  "description": "作品简介",
  "story": "延伸说明",
  "image": "assets/images/东坡夜游.jfif"
}
```

### 4. `craftSteps`

用于“工艺解构”页面。

每个步骤包含：

- `title`：步骤名称
- `summary`：简短说明
- `detail`：详细介绍
- `focus`：阅读焦点

## 新增或修改展品

如果你要新增一个展品，按下面步骤处理：

1. 把新图片放到 `assets/images/`
2. 在 `assets/data/content.json` 的 `exhibits` 数组中新增一项
3. 为新展品设置唯一 `id`
4. 检查 `image` 路径与图片文件名完全一致
5. 重新运行项目确认显示效果

## 替换图片时的注意事项

- 优先使用清晰、裁切合理的图片
- 文件名尽量保持简洁且可读
- 如果你使用中文文件名，更新 JSON 时必须完全一致
- 如果图片加载失败，界面会显示默认占位图标

## 分类字段建议

当前项目里常见分类包括：

- `人物`
- `山水`
- `龙凤`
- `花鸟`

如果新增分类，应用会自动把它加入筛选列表，不需要修改额外代码。

## 文案修改建议

为了保持项目整体风格，建议文案遵循以下原则：

- 句子不要太长，适合移动端阅读
- 说明尽量有画面感，避免纯百科式表述
- 保持“新中式极简”和“非遗数字化展示”的整体语气
- `summary` 适合一句话概括
- `detail` 可以写得更完整
- `focus` 尽量写成“观众看什么”的提示语

## 修改完成后的验证步骤

每次改完内容后，建议执行：

```bash
flutter pub get
flutter run -d chrome
```

至少检查以下内容：

1. 引导页是否正常显示
2. 展馆列表是否正常加载
3. 新增或替换图片是否能显示
4. 工艺步骤弹层内容是否完整
5. 没有出现 JSON 格式错误

## 常见错误

### 1. JSON 逗号写错

最常见的问题是最后一项后面多了逗号，或者新增字段时少了逗号。

### 2. 图片路径写错

例如文件名是 `东坡夜游.jfif`，但 JSON 写成了 `东坡夜游.jpg`，就会导致加载失败。

### 3. `id` 重复

如果两个展品使用相同 `id`，收藏、最近浏览等逻辑可能出现混乱。

## 推荐维护流程

建议每次内容更新按以下顺序进行：

1. 先准备图片和文案
2. 再更新 `content.json`
3. 本地运行检查
4. 提交到 GitHub
5. 如需对外展示，再重新发布 APK 或 Web 原型
