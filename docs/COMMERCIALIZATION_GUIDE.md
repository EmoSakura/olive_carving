# 商用化推进建议

这份文档用于说明榄雕云艺项目从“课程原型”走向“可对外演示、可逐步商用”的推荐路径。

## 当前已经具备的基础

- 真实注册登录入口
- 产品首页与更完整的信息架构
- 管理员后台入口
- Supabase 认证与后台状态读写能力
- 本地与云端双模后端结构
- 本地用户状态持久化：收藏、最近浏览、学习进度、互动作品档案
- Android Release 构建能力
- 正式签名接入口：`android/key.properties.example`

## 距离真正商用还差什么

### 1. 正式品牌资产

- 替换默认图标、启动图、应用名和品牌视觉
- 明确品牌口号、项目介绍和对外展示话术
- 统一图片版权来源与授权说明

### 2. 正式签名与分发

- 生成正式 keystore
- 在本地创建 `android/key.properties`
- 使用正式签名重新构建 APK / AAB
- 准备版本号规范和更新说明

### 3. 后台与数据能力

当前版本已经有真实 Supabase 认证与后台状态读写能力，但仍处于“产品雏形”阶段。

如果要进入长期商用，建议优先补这三块：

1. 用户行为数据上云
2. 内容后台或 CMS
3. 管理后台的正文 / 图片 / 标签编辑能力

### 4. 合规与商用材料

- 隐私政策
- 用户协议
- 图片 / 文字版权说明
- 商务合作联系方式
- 对外报价单与服务边界说明

## 推荐的下一阶段路线

### 阶段一：演示级商用

目标：用于提案、路演、客户演示、课程展示。

优先做：

1. 正式图标与品牌视觉
2. 正式 Android 签名
3. 更多真实内容与高清素材
4. 打包一份演示账号和展示脚本

### 阶段二：轻量业务闭环

目标：让账号、后台和用户数据真正跑起来。

优先做：

1. 收藏、学习进度、互动归档同步到 Supabase
2. 内容管理从本地 JSON 迁移到数据库
3. 扩展管理员后台，支持在线编辑
4. 增加访问统计与转化统计

### 阶段三：正式产品化

目标：可长期运营、可接项目、可稳定迭代。

优先做：

1. 完整账号系统与角色权限
2. 云端内容审核与发布流程
3. Web 正式部署域名
4. 法务文档与合同模板
5. 商店上架材料

## 建议的商业切入口

当前项目最适合优先切入：

- 学校研学课程
- 地方文旅数字导览
- 博物馆 / 展馆专题展示
- 品牌联名文化活动

原因是这些场景对“真实账号入口 + 内容展示 + 互动体验 + 后台可控发布”的组合最敏感，也最容易把现有项目转化成可成交的数字方案。
